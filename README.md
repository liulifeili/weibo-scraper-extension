# 微博博主主页爬取工具

一款专为**Edge浏览器**设计的微博爬虫工具，自动注入到微博页面左下角，支持自动翻页抓取博主主页的所有微博数据，并导出为CSV文件。

![Version](https://img.shields.io/badge/version-v2.0-blue)
![Edge](https://img.shields.io/badge/Edge-Supported-green)

---

## 📌 功能特性

- 🔄 **自动翻页** - 智能点击微博页面翻页按钮，自动加载更多内容
- 📊 **数据完整** - 支持转发数、评论数、点赞数等互动数据
- 🛡️ **智能去重** - URL去重机制，防止重复抓取，自动检测循环
- ⏸️ **暂停/继续** - 支持随时暂停和继续爬取
- 📥 **CSV导出** - UTF-8编码，兼容Excel直接打开
- 📅 **日期转换** - 自动将微博日期格式转换为标准格式
- 🎨 **精美UI** - 渐变橙色主题，流畅动画效果
- 📍 **左下角显示** - 自动注入到页面左下角，不遮挡其他扩展

## 💿 安装说明

> ⚠️ **重要提示**：本扩展**未上架Edge应用商店**，需要手动加载

### 环境要求

- Microsoft Edge 浏览器（Chromium内核）
- 启用开发者模式

### 安装步骤

#### 1. 下载/克隆项目

```bash
git clone https://github.com/yourusername/weibo-scraper-extension.git
```

或者直接下载ZIP文件并解压。

#### 2. 打开Edge扩展页面

在Edge浏览器地址栏输入：
```
edge://extensions/
```

#### 3. 启用开发者模式

页面右上角找到 **"开发者模式"** 开关，将其 **开启**

#### 4. 加载扩展

1. 点击 **"加载解压缩的扩展"** 按钮
2. 选择项目文件夹 `weibo_scraper_extension`
3. 确认加载

#### 5. 完成安装

扩展加载成功后，会在扩展列表中看到 **"微博博主主页爬取工具"**

## 🚀 使用方法

### 1. 打开微博博主主页

在浏览器中访问目标博主的微博主页，例如：
```
https://weibo.com/u/6251584258
```

### 2. 自动显示工具条

页面加载完成后，工具条会自动出现在**左下角**，无需点击任何按钮。

### 3. 开始爬取

1. URL输入框会自动填充当前页面URL
2. 点击 **▶ 开始爬取** 按钮
3. 观察实时进度和日志

### 4. 暂停/继续

爬取过程中可以随时点击 **⏸ 暂停** 按钮暂停，之后可点击 **▶ 继续** 恢复爬取。

### 5. 下载数据

爬取完成后，点击 **⭳ 下载CSV** 按钮，CSV文件会自动下载到默认下载目录。

## ⚙️ 工作原理

### API请求流程

微博使用API进行数据加载，工具自动调用以下API：

**第1页请求：**
```
https://weibo.com/ajax/statuses/mymblog?uid={UID}&page=1&feature=0
```

**第2页及后续请求：**
```
https://weibo.com/ajax/statuses/mymblog?uid={UID}&page={N}&feature=0&since_id={上一页最后一条ID}
```

### 翻页机制

- 自动查找并点击页面中的"加载更多"按钮
- 按钮样式：`div.woo-box-flex.woo-box-alignCenter.woo-box-justifyCenter._nextPage_`
- 等待内容加载完成后自动获取下一页数据

### 去重机制

- 记录所有已请求的URL
- 检测 `since_id` 重复
- 连续2次重复自动停止，防止无限循环

## 🔧 技术细节

### 项目结构

```
weibo_scraper/
├── manifest.json       # Edge扩展配置文件
├── content.css        # 样式表
├── content.js         # 核心逻辑（注入+爬取）
├── icons/             # 扩展图标
│   ├── icon16.png
│   ├── icon48.png
│   └── icon128.png
└── README.md          # 本文件
```

### 技术栈

- **前端**：原生 HTML/CSS/JavaScript
- **数据获取**：Fetch API
- **数据格式**：JSON → CSV
- **浏览器**：Microsoft Edge (Chromium)

### manifest.json 配置

```json
{
  "manifest_version": 3,
  "name": "微博爬虫工具",
  "version": "2.0",
  "description": "微博爬虫工具，自动注入到页面左下角，支持自动翻页、URL去重",
  "permissions": ["downloads"],
  "host_permissions": ["https://weibo.com/*"],
  "content_scripts": [{
    "matches": ["https://weibo.com/*"],
    "css": ["content.css"],
    "js": ["content.js"],
    "run_at": "document_idle"
  }]
}
```

### 权限说明

- `downloads`：允许下载CSV文件
- `https://weibo.com/*`：允许访问微博域名

## ⚠️ 注意事项

### 使用须知

1. **登录状态**
   - 确保微博处于登录状态，否则可能无法获取完整数据
   - 部分用户数据可能需要关注后才能查看

2. **网络稳定**
   - 爬取过程中保持网络连接
   - 中断可能导致数据不完整

3. **耐心等待**
   - 大规模爬取需要较长时间
   - 请勿关闭浏览器窗口
   - 建议单次爬取不超过5000条

4. **合理使用**
   - 不要过于频繁地爬取
   - 遵守微博平台规则
   - 仅供学习研究使用

### 数据完整性

- 部分被删除或不可见的微博无法爬取
- 转发内容可能只显示原始微博ID
- 图片和视频不包含在CSV中

## ❓ 常见问题

### Q: 工具条没有出现？

**检查步骤：**
1. 确认扩展已正确加载
2. 刷新微博页面
3. 检查是否在微博域名页面（weibo.com/*）

### Q: 提示"无法解析UID"？

**原因：** 输入的不是微博用户主页URL

**解决：** 确保输入类似 `https://weibo.com/u/6251584258` 的格式

### Q: 爬取到一半停止了？

**可能原因：**
- 网络不稳定 → 等待后自动重试
- 触发反爬机制 → 间隔几分钟后重试
- 已全部加载完成 → 正常停止

### Q: CSV文件打开乱码？

**解决方案：**
- Excel导入时选择 UTF-8 编码
- 使用 WPS 打开，自动识别编码
- 用 Notepad++ 打开后另存为 ANSI 编码

### Q: 页面翻页按钮找不到？

**说明：** 
- 工具会尝试多种选择器定位翻页按钮
- 如果按钮被CSS隐藏，工具会尝试滚动到可见位置
- 部分页面可能没有翻页按钮（只有一页）

## 🛠️ 故障排除

### 扩展加载失败

1. 检查 `manifest.json` 是否存在且格式正确
2. 确认所有必需文件都在同一目录
3. 查看Edge扩展页面的错误提示

### 控制台调试

1. 在微博页面按 **F12** 打开开发者工具
2. 切换到 **Console** 标签
3. 查找 `[微博爬虫]` 开头的日志

### 扩展重载

1. 打开 `edge://extensions/`
2. 找到 "微博博主主页爬取工具"
3. 点击 **刷新按钮** 🔄
4. 刷新微博页面
